🔌 Connect Your Supabase

Paste your Project URL and Anon Key. Saved in browser. Follow the steps below to get them.

1
Create a Free Supabase Account
Takes 2 minutes, completely free
  1. Go to supabase.com and click "Start your project"
  2. Sign up using GitHub or email — both are free
  3. After login, click "New project" (green button)
  4. Enter Project name: foodbridge
  5. Enter a strong database password (save it somewhere!)
  6. Choose Region → pick Southeast Asia (Singapore) for India
  7. Click "Create new project" → wait about 60 seconds ⏳
2
Enable Email Auth
So users can sign up and log in
  1. In your Supabase project, click Authentication in the left sidebar
  2. Click Providers
  3. Find Email — it should already be enabled (green toggle)
  4. Scroll down and disable "Confirm email" (turn it OFF for hackathon demo — so users don't need to verify email)
  5. Click Save
3
Run the SQL to Create Tables
Creates all 3 tables your app needs
  1. In your project, click SQL Editor in the left sidebar
  2. Click "New query" button
  3. Copy the SQL below and paste it into the editor
  4. Click the green "Run" button (or press Ctrl + Enter)
  5. You should see "Success. No rows returned" ✅
-- ═══ TABLE 1: user_profiles (buyer/seller info) ═══
CREATE TABLE user_profiles (
  id          UUID REFERENCES auth.users(id) PRIMARY KEY,
  name        TEXT NOT NULL,
  phone       TEXT,
  email       TEXT,
  address     TEXT,
  upi_id      TEXT,
  role        TEXT DEFAULT 'buyer',
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- ═══ TABLE 2: foods (listings) ═══
CREATE TABLE foods (
  id             UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id        UUID REFERENCES auth.users(id),
  title          TEXT NOT NULL,
  quantity       TEXT NOT NULL,
  location       TEXT NOT NULL,
  contact        TEXT,
  description    TEXT,
  category       TEXT DEFAULT 'other',
  listing_type   TEXT DEFAULT 'free',
  original_price NUMERIC(10,2),
  discount_pct   INTEGER,
  final_price    NUMERIC(10,2),
  upi_id         TEXT,
  expiry_time    TIMESTAMPTZ NOT NULL,
  status         TEXT DEFAULT 'available',
  created_at     TIMESTAMPTZ DEFAULT NOW()
);

-- ═══ TABLE 3: orders ═══
CREATE TABLE orders (
  id            UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  food_id       UUID REFERENCES foods(id),
  buyer_id      UUID REFERENCES auth.users(id),
  buyer_name    TEXT NOT NULL,
  buyer_phone   TEXT NOT NULL,
  buyer_email   TEXT,
  buyer_address TEXT,
  quantity_req  TEXT,
  message       TEXT,
  order_type    TEXT DEFAULT 'claim',
  amount        NUMERIC(10,2) DEFAULT 0,
  status        TEXT DEFAULT 'pending',
  ack_code      TEXT,
  seller_note   TEXT,
  created_at    TIMESTAMPTZ DEFAULT NOW()
);

-- ═══ Row Level Security (open for hackathon) ═══
ALTER TABLE user_profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE foods          ENABLE ROW LEVEL SECURITY;
ALTER TABLE orders         ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Public read/write" ON user_profiles FOR ALL USING (true) WITH CHECK (true);
CREATE POLICY "Public read/write" ON foods          FOR ALL USING (true) WITH CHECK (true);
CREATE POLICY "Public read/write" ON orders         FOR ALL USING (true) WITH CHECK (true);
4
Get Your Project URL & API Key
You'll paste these into the box at the top of this page
  1. In your project, click Project Settings (gear icon ⚙️ at the bottom of left sidebar)
  2. Click API in the settings menu
  3. Under "Project URL" — copy the URL (looks like https://abcdef.supabase.co)
  4. Under "Project API keys" — copy the anon / public key (starts with eyJ...)
  5. Paste both values in the "Connect Your Supabase" box at the top of this page
  6. Click 💾 Save & Test Connection
5
Test It!
Make sure everything is working
  1. After saving, the app will say "✅ Connected!" at the top
  2. Log out and create a new account using the signup form
  3. Try adding a food listing from the ➕ Add Listing tab
  4. Open your Supabase dashboard → Table Editor → foods — you should see your listing appear there!
  5. Now try placing an order on Browse tab — check Table Editor → orders
✅ All done! Your app is now fully connected to a live database. Data persists across sessions and devices.
